# 導生密碼查詢系統 — 部署與使用教學

這份教學從零開始，一步一步帶你把系統架起來。**完全沒寫過程式也能照做。**
做好之後，每學期更新只要改一個 Google 試算表就好。

---

## 這套系統由三個東西組成

1. **Google 試算表** — 放全班名單（私有，只有你看得到）。
2. **查詢程式（Apps Script）** — 附在試算表上的小程式，學生查詢時它**只回傳一筆**結果。
3. **兩個網頁檔**（放到學校網頁空間）：
   - `index.html` — 學生看到的查詢頁（這個檔**不用改**）。
   - `config.js` — 小小的設定檔（**只改這個**：填查詢網址、改標題）。

> 為什麼安全：名單只存在 Google 試算表裡，學生拿不到；程式只回傳「比對成功的那一筆」，全班名單永遠不會送到瀏覽器，學校網頁上也**沒有任何學生資料**。

---

## 第 1 步：建立試算表並貼上名單

1. 用電腦瀏覽器（建議 **Google Chrome**）登入要使用的 Google 帳號，開 <https://sheets.google.com> 新增一份試算表。
2. 把左下角的分頁名稱改成 **`名單`**（兩個字，不要有空白）。
3. 第一列輸入標題，**名稱要一模一樣**：

   | 系級 | 學號 | 姓名 | 115上導師密碼 | 就學狀態 |
   |---|---|---|---|---|

   - 密碼那一欄的標題只要**包含「密碼」**兩字即可（`115上導師密碼`、`115下導師密碼` 都行）。
4. 把名單內容從第二列開始貼進去。
5. 建議把「學號」「密碼」兩欄設成純文字：選取整欄 →「格式 → 數值 → 純文字」，避免長數字被改格式或掉開頭的 0。

---

## 第 2 步：加入查詢程式

> ⚠ 這一步請**務必用 Google Chrome**。Edge、Brave、Safari 開 Apps Script 常會出現「無法開啟檔案」，那是瀏覽器的隱私設定擋住，不是檔案壞掉。

1. 在試算表上方選單點：**擴充功能 → Apps Script**。
2. 把編輯器裡預設的內容**全部刪掉**，貼上 `Code.gs` 的內容。
3. 按上方的 💾 儲存。

---

## 第 3 步：部署，取得查詢網址

1. 編輯器右上角：**部署 → 新增部署作業**。
2. 左上齒輪選擇類型 → **網頁應用程式**。
3. 兩個設定請照填：
   - **執行身分**：`我`
   - **誰可以存取**：`任何人`　←（最容易設錯，一定要選「任何人」，學生才不用登入）
4. 按**部署**。第一次會要授權：
   - 看到「Google hasn't verified this app / 尚未驗證」是**正常的**（因為這是你自己剛建的程式）。
   - 點 **Advanced（進階）→ Go to…（前往專案，不安全）→ Allow（允許）**。
5. 完成後會出現「網頁應用程式」的**網址**，是一串 `https://script.google.com/macros/s/.../exec`（結尾 **`/exec`**）。
   - 注意有兩個「複製」鈕：要按**「網址」**那個，不要按上面的「部署作業 ID」。
6. 把這串網址**複製起來**，下一步要用。

---

## 第 4 步：填設定檔 `config.js`

用記事本（或任何文字編輯器）打開 `config.js`，你只會看到幾行。照下面改：

```js
const CONFIG = {
  endpoint: "貼上你的_exec_網址",   // ← 換成第 3 步複製的 /exec 網址
  titleZh: "小樽導生密碼查詢",        // ← 想改標題就改這裡
  titleEn: "Advising Password",
  accentColor: "#82318e",            // ← 顏色可不改；清華紫
};
```

- 引號 `" "`、每行結尾的逗號都要保留，不要刪。
- `index.html` **完全不用動。**
- 改完存檔。

---

## 第 5 步：上傳到清大教職員工網頁空間

這一步是把 `index.html` 和 `config.js` 兩個檔，透過 FTP 軟體傳到學校主機。

### 5-1　先認識你的網址

- 教職員工網頁主機是 **mx.nthu.edu.tw**。
- 上傳後，你的查詢頁網址會是：**`http://mx.nthu.edu.tw/~你的帳號/`**
  （把「你的帳號」換成你的學校信箱帳號）。
- 系統會自動讀取你帳號底下 **`WWW`** 資料夾裡的 `index.html`。

### 5-2　安裝 FileZilla（傳檔軟體）

學校示範用免費軟體 **FileZilla**。安裝教學見學校頁面：
<https://net.nthu.edu.tw/netsys/ftp:install:filezilla>

### 5-3　連線到主機

打開 FileZilla，上方有四個欄位，照下面填，然後按「**快速連線**」：

| 欄位 | 要填的內容 |
|---|---|
| 主機(H) | `mx.nthu.edu.tw` |
| 使用者名稱(U) | 你的學校信箱**帳號** |
| 密碼(W) | 你的學校信箱**密碼** |
| 連接埠(P) | `21`（可留空） |

> 想固定存起來下次直接用，可改用「站台管理員 → 新增站台」，登入型式選「**一般**」，其餘相同。
> 參考學校說明：<https://net.nthu.edu.tw/netsys/ftp:connect>

### 5-4　上傳兩個檔

1. 連上後，畫面**右半邊**是學校主機裡的檔案。
2. 找到名為 **`WWW`** 的資料夾，**點兩下進去**。
   - 如果沒有這個資料夾：在右邊空白處按右鍵 →「建立目錄」，命名為 `WWW`（**大寫**），再進去。
3. 畫面**左半邊**是你自己的電腦，找到 `index.html` 和 `config.js`。
4. 把這**兩個檔**一起選起來，拖曳到右邊（WWW 資料夾內）即可上傳。

### 5-5　確認成功

打開瀏覽器，輸入 **`http://mx.nthu.edu.tw/~你的帳號/`**，看到查詢頁、試查一筆能查到密碼，就成功了 🎉

---

## 每學期更新（最簡單）

**不用碰程式、不用重新上傳。** 只要：

1. 打開那份 Google 試算表的 `名單` 分頁。
2. 把密碼那一欄換成新學期的值（也可把標題改成 `115下導師密碼`）。
3. 名單有人異動就直接增刪該列。

存檔後**即時生效**。

> 想換標題或顏色：改 `config.js` 後，用 FTP 重新上傳那一個檔即可（試算表不用動）。

---

## 內建的安全機制

- **只回單筆**：沒有「列出全部」的功能；姓名 + 學號**兩者都對**才回傳。
- **單一學號鎖定**：同一學號連續錯 5 次 → 鎖 1 小時。
- **全站節流**：每 10 分鐘最多 80 次查詢，擋大量掃描。
- **查詢紀錄**：每次查詢自動寫進 `查詢紀錄` 分頁（時間／學號／姓名／結果）。
- **統一錯誤訊息**：查不到時不分「學號錯」或「姓名錯」，避免洩漏線索。

要調整數字，改 `Code.gs` 最上面的設定區（改完要重新部署才生效）。

## 疑難排解

- **點 Apps Script 出現「無法開啟這個檔案」**
  改用 **Google Chrome**；或在 Edge/Brave 把「封鎖協力廠商 Cookie」關掉、Brave 的盾牌(Shields)對該網站關閉。

- **查詢頁顯示「連線失敗」**
  確認 `config.js` 裡的 `endpoint` 是 `/exec` 結尾、且部署時「誰可以存取」設成「**任何人**」。

- **網頁打不開 / 看到不能瀏覽**
  檢查：資料夾要大寫 **`WWW`**、檔名要小寫 **`index.html`**、`index.html` 和 `config.js` 要放在**同一個** WWW 資料夾裡。

- **「複製密碼」按鈕**
  學校網頁是 http，瀏覽器會自動改用相容方式複製，功能正常，可放心。

- **資料安全**
  學校網頁空間有提醒「請審慎評估放置資料」。本系統放到學校主機的只有 `index.html` 與 `config.js`，**裡面沒有任何學生名單或密碼**；真正的資料都在你私有的 Google 試算表，不會出現在學校主機上。

## 分享給同事

請同事各自完成「第 1～5 步」即可擁有自己的系統（各自的試算表、各自的查詢網址、各自的學校網頁）。他們平常只需要：

- 改標題／顏色 → 動 `config.js`
- 換每學期資料 → 動 Google 試算表
